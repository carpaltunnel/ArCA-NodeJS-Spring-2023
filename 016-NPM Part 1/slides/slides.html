NPM (Node Package Manager)







NPM gives us an easy way to create, install, manage, and share code. With over one million packages, it's one of the largest sources of third party libraries in the world.

npm init

npm init is used to initialize a new NPM project. The initialization script will interactively ask for information about your new project, though all of it is optional. Once complete, a package.json will be created.

package.json

package.json is the core of a NPM project and defines dependencies, versions, and tasks.

{
    "name": "testing-npm",
    "version": "0.0.1",
    "description": "Just a test",
    "main": "index.js",
    "dependencies": {
        "lodash": "^4.17.21"
    },
    "devDependencies": {}
}

npm install

Executing npm install package-name will download "package-name" from NPM, install it locally, and add it to the package.json dependencies list. Be sure to type the package name correctly...

node_modules

After running npm install, the package will be installed into a local folder named node_modules. You can dig around in this directory for research and debugging but you should never commit it.

dependencies & devDependencies

When you install a dependency it is automatically added to the package.json "dependencies" object. You can also install a "development" dependency with npm install --save-dev package-name. This will add the package to the "devDependencies" object of package.json.

Global Installs

While not super common, you can also install NPM packages globally / system-wide. npm install -g package-name will install the package for the entire system. This is most commonly used for utility libraries like Grunt.

Dependency Versions

  1. ^ : Accept any version update that does not change the leftmost non-zero number.
  2. ~ : Accept only "patch" updates (1.1.*)
  3. > : Accept any version higher than the one specified
  4. >= : Accept any version equal to or higher than the one specified
  5. <= : Accept any version equal or lower to the one specified
  6. < : Accept any version lower than the one specified
  7. = : Accept the exact version and nothing else
  8. - : Accept a range of versions. Example: 1.1.0 - 1.5.5
  9. || : Accept a combination of sets (logical OR). Example: < 1.0 || > 1.5

package-lock.json

package-lock.json is automatically generated by NPM whenever node_modules or package.json are changed. It documents the exact versions of all dependencies (and their dependencies) that were installed so subsequent builds by other users can be exactly the same. You should commit package.json and package-lock.json to your Git repo.

Git Dependencies

If you have an internal dependency that can't be published to NPM, you can reference a Git repo directly.

"my-package": "git+https://my-git-repo.mydomain.com/projects/my-package#v1.1"
